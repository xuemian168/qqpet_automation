# 安全策略

本项目是个人维护的怀旧研究项目，没有商业 SLA，但安全问题会认真处理。

## 报告漏洞

**不要在 public issue 里直接贴漏洞细节。** 走下面任一渠道：

- GitHub Security Advisory：仓库 → Security → Report a vulnerability
- 邮箱：通过仓库主页公示的联系方式

报告里给到这些就够了：

- 受影响的组件（Electron 端 / Python CLI / 构建产物 / CI 工作流）
- 受影响的版本（commit hash 或 release tag）
- 复现步骤、最小化 PoC
- 影响范围（本地 RCE / 信息泄漏 / 提权 / 等等）
- 你希望的署名方式（如果想被致谢）

收到后我会在 **7 天内** 给初步回复。修复时间取决于严重程度，没有承诺。

## 披露原则

- 在补丁发布前，请不要公开漏洞细节
- 修复发布后，会在 release notes 里致谢报告者（除非你要求匿名）
- 如果 30 天内没收到我的回复，你可以选择公开披露——这是合理做法

## 支持的版本

只有最新 release 会收到安全修复。老版本请升级。

| 版本 | 状态 |
|---|---|
| 最新 release | 接受修复 |
| 历史版本 | 不接受 |
| `main` 未发布提交 | 滚动修复 |

## 范围内 / 范围外

**算漏洞：**

- 本地 RCE、提权、沙箱逃逸
- 通过恶意 `config-macos.json` 触发的 Electron 端崩溃或代码执行
- Python CLI 解析用户输入时的注入、路径穿越
- 构建产物、CI 工作流被注入的供应链风险
- 硬编码的真实凭据、token 泄漏

**不算漏洞，请直接提 Issue：**

- 应用未签名导致的 macOS Gatekeeper / Windows SmartScreen 提示——这是已知现状，README 已说明
- "应用没连服务器" / "登录不上"——腾讯官方服务器早已停服，本项目仅本地运行，不会回连任何线上服务
- 第三方依赖（Electron、Chromium、Ruffle、npm/PyPI 包）的已知 CVE——请去对应上游报告；如果本项目能通过升级缓解，可以提 PR 或 Issue
- 反编译、修改本项目以达成恶意目的——这属于使用者自身行为，不在威胁模型内

## 不在范围内

不要拿本项目去做下面这些事，发现了也别报到这里——这不是漏洞，是滥用：

- 攻击腾讯任何在运营服务
- 伪造客户端身份对接非怀旧停服环境
- 把本项目改成代练、外挂、广告、付费墙再分发
